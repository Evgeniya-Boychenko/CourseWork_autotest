# План автоматизации тестирования сервиса покупки туров

## 1. Перечень автомтизмируемых сценариев
Автоматизация формы: **"Оплата по дебетовой карте"**

Будут автоматизированы сценарии:

**Позитивные сценарии:**
1. Успешная оплата тура с одобренной картой (1111 2222 3333 4444).
Проверка сообщения об успешной операции.

**Негативные сценарии:**
1. Оплата тура с отклоненной картой (5555 6666 7777 8888). Проверка сообщения об ошибке.
2. Ввод невалидного номера карты (с большим или меньшим количеством цифр). Проверка сообщения о неверном формате.
3. Ввод карты с истекшим сроком действия (дата в прошлом). Проверка сообщения об истечении срока действия карты.
4. Ввод невалидного значения в поле "владелец" (цифры или спецсимволы). Проверка сообщения о неверном формате.
5. Попытка отправки незаполненной формы. Проверка сообщения о необходимости заполнить форму.
6. Ввод невалидного CVC/CVV (буквы, менее 3 символов). Проверка сообщения о неверном формате.

**Перечень используемых инструментов с обоснованием выбора**

1. **Python** - язык, который я изучала.
2. **pytest** - фреймворк для запуска тестов.
3. **Selenium WebDriver** - фреймворк для автомиатизации  веб-тестирования, который позволяет имитировать действия пользователя в браузере
4. **MySQL** - система управления БД.
5.  **Allure Report** - инструмент для генерации визуальных HTML отчетов.
6. **Docker + Docker Compose** - для развертывания изолированного экземлпляра БД MySQL
7. **GitHub** - Публичная система контроля версий для хранения кода, документации и истории изменений.
