# Результаты тестирования

## Краткое описание

- Проведено тестирование учебной формы для покупки туров. Тестировалась функция оплаты дебетовой картой. Проверялась: валидация полей, сообщения об успехе/ошибке, сохранение статуса в БД.
- В ходе тестирования использовались инструменты: PyMySQL, Docker + Docker Compose, Java, Selenium, pytest, allure-pytest.
- Всего написано 16 тестов.

## Статистика

- Всего тестов: 16
- Успешных: 15
- Ожидаемо падающих (xfail): 1
- Процент успешных: 94.1%

## Расхождение с планом тестирования

- В ходе тестирования выявлена невозможность точно следовать написанным в плане сценариям, в связи с реальным поведением системы.

## Найденные баги

### Баг №1: Отсутствует валидация месяца 00.
**Серьёзность:** Major (часть основной бизнес-логики работает некорректно)
**Описание:** Форма принимает значение "00" в поле "Месяц" и успешно отправляется, хотя такого месяца не существует.
**Шаги воспроизведения:** 
1. Ввести в поле "Месяц" значение 00.
2. Заполнить остальные поля валидными данными.
3. Нажать "Продолжить".
**Ожидаемый результат:** Появляется сообщение "Неверный формат" под полем месяца, форма не отправляется.
 **Фактический результат:** Форма отправляется, появляется уведомление "Успешно! Операция одобрена Банком"

### Баг №2: Отсутствует валидация имени владельца.
**Серьёзность:** Major (часть основной бизнес логики работает некорректно).
**Описание:** Поле "Владелец" принимает любые символы (кириллицу, цифры, спецсимволы, одну букву, слишком длинное имя), хотя должно принимать только латинские буквы.
**Шаги воспроизведения:** 
1. Ввести в поле "Владелец" невалидное значение (данные кириллицей: "Иван Иванов", цифры "123", спецсимволы "!@#$%", одну букву "а", ввести длинное имя ivan ivanov ... 100+ символов).
2. Заполнить остальные поля валидными данными.
3. Нажать "Продолжить"
**Ожидаемый результат:** Появляется сообщение "Неверный формат" под полем владельца, форма не отправляется.
**Фактический результат:** Форма успешно отправляется, появляется уведомление "Успешно! Операция одобрена Банком".

### Баг №3: Симулятор банка одобряет DECLINED карту.
**Серьёзность:** Critical(неправильно работает ключевая бизнес-логика)
**Описание:** Карта с номером 5555 6666 7777 8888 должна быть отклонена банком, но симулятор банка одобряет её.
**Шаги воспроизведения:**
1. Ввести в поле "Номер карты" значение "5555 6666 7777 8888".
2. Заполнить остальные поля валидными данными.
3. Нажать "Продолжить".
**Ожидаемый результат:** Появляется сообщение "Ошибка! Операция отклонена Банком", в базе данных статус платежа DECLINED.
**Фактический результат:** Появляется сообщение "Успешно! Операция одобрена Банком", в базе данных статус платежа APPROVED.

### Баг №4: Некорректное отображение ошибки при пустом CVC
**Серьёзность:** Minor (не нарушает бизнес-логику, ошибка UI)
**Описание:** При отправке формы с пустым полем CVC сообщение об ошибке "Поле обязательно для заполнения" появляется под полем "Владелец", хотя поле заполнено корректно.
**Шаги воспроизведения:**
1. Открыть форму оплаты
2. Заполнить поля валидными данными (карта: 1111 2222 3333 4444, месяц: 06, год: 27, владелец: ivan ivanov)
3. Оставить поле CVC пустым
4. Нажать "Продолжить"
**Ожидаемый результат:** Сообщение "Неверный формат" появляется только под полем CVC.
**Фактический результат:** Сообщение "Поле обязательно для заполнения" появляется под полем "Владелец", хотя поле заполнено и сообщение "Неверный формат" появляется под полем CVC.

## Рекомендации

2. Исправить валидацию поля "Владелец" - поле должно принимать только латинские буквы и пробелы, блокировать кириллицу, цифры, спецсимволы.
3. Настроить прием только одобренной карты - карта 5555 6666 7777 8888 должна возвращать статус DECLINED/

5. Исправить ошибку с появлением сообщения об ошибке, которой нет под полем "Владалец", при отправке формы с незаполненным CVC.
