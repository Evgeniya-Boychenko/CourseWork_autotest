#Отчет от автоматизации

**Репозиторий:** [CourseWork_autotest](https://github.com/Evgeniya-Boychenko/CourseWork_autotest)

## 1.Было запланировано

В проекте была поставлена цель автоматизировать тестирование формы оплаты по дебетовой карте.
- Проверка UI (сообщения об успехе/ошибке, валидация полей).
- Проверка БД (статусы APPROVED/DECLINED).
- Генерация отчетов Allure.

## 2. Реализовано
Архитектура проекта - создана структура (pages/, tests/, conftest.py, requirements.txt).  
Page Object - создан класс DebitFormPage с локаторами и методами взаимодействия с формой.  
UI-тесты - написано 16 автотестов, покрывающих позитивные и негативные сценарии для всех полей формы.  
Тесты с проверкой БД - реализованы сценарии, которые после отправки формы, подключаются к MySQL через PyMySQL и проверяют актуальный статус платежа.  
Allure - настроена генерация HTML-отчётов.  
Документация - подготовлены README.md, Plan.md, Report.md, Summary.md  


## 3. Что не реализовано и причины

Автоматизация сценариев с вводом букв, спецсимволов и превышением лимита символов в полях "Номер карты", "Месяц", "Год", "CVC".
Причина: поля имеют встроенные HTML-маски, которые блокируют ввод недопустимых символов на уровне браузера. 

## 4. Cработавшие риски и ограничения
Ограничения UI-компонентов: HTML-маски полей сократили покрытие негативных сценариев для полей "Карта", "Месяц", "Год", "CVC".  
Сработавшие риски:
Обнаружен баг симулятора банка (DECLINED карта ошибочно одобряется). Тест "test_decline_card_with_db_check" падает, выявляя этот дефект. Баг заведён в GitHub Issues: [ссылка](https://github.com/Evgeniya-Boychenko/CourseWork_autotest/issues/3).
Зависимость от окружения: без запуска Docker-контейнера с MySQL и запущенного приложения aqa-shop.jar тесты не запускаются. 
UI-баг с отображением ошибок: При пустом поле CVC сообщение об ошибке дублируется под полем "Владелец". Это косметическая проблема, не влияющая на функциональность.
